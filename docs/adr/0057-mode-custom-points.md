# ADR-0057: モードカスタムポイントと5層設定解決

## ステータス

提案

## コンテキスト

ADR-0017 で設定レジストリ（SettingsRegistry）とバッファローカル設定（BufferLocalSettings）を導入した。
現在の設定解決は「バッファローカル値 → Setting.defaultValue」の2層のみであり、モードが設定デフォルト値を持つ仕組みがない。

Emacsではモードフックがバッファローカル変数を設定することでモード固有の設定を実現しているが、これは命令的なアプローチである。
本エディタではモードが宣言的にデフォルト値を保持できる仕組みを導入し、同等の効果をより明示的に得られるようにしたい。

また、SettingsRegistryにはユーザーが明示的に設定したグローバル値のストアがなく、Setting.defaultValueしか保持していない。
ユーザーが `set-variable` 等でグローバルに値を変更するユースケースに対応するため、グローバル値ストアの追加も必要である。

## 決定

設定値の解決に以下の5層の優先順位を導入する。

1. **バッファローカル変数** — ユーザーが特定バッファに対して明示的に設定した値
2. **マイナーモードデフォルト** — 有効なマイナーモードが持つデフォルト値（後から有効にしたものが優先）
3. **メジャーモードデフォルト** — バッファのメジャーモードが持つデフォルト値
4. **エディタグローバル変数** — ユーザーが明示的に設定したグローバル値
5. **エディタデフォルト値** — Setting定義時のdefaultValue

### モードのデフォルト値保持

ModeSettingsクラスを新規導入する。モード固有のデフォルト値マップをイミュータブルに保持する。
MajorMode、MinorModeの両インターフェースに `default ModeSettings settingDefaults()` メソッドを追加する。
デフォルト実装は空のModeSettingsを返すため、既存のモード実装への影響はない。

### SettingsRegistryの拡張

ユーザーグローバル値のストアを追加する。
`getGlobal(Setting<T>)` は `Optional<T>` を返し、フォールバックは含めない。
フォールバックの責務はBufferLocalSettingsが持つ。

### マイナーモードの優先順位

キーマップ解決（CommandLoop.resolveKey）と同じく、後から有効にしたマイナーモードが優先する。
リストの末尾から逆順に探索し、最初に見つかった値を採用する。

## 結果

- モードが宣言的にデフォルト値を保持でき、モードフック相当の機能が実現される
- 5層の明確な優先順位により、設定値の出所が追跡しやすくなる
- 既存のモード実装（TextMode、MarkdownMode）はdefaultメソッドにより変更不要
- BufferLocalSettingsのコンストラクタは破壊的変更となるが、本体コードでの使用箇所がないため影響はテストに閉じる
