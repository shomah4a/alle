# ADR-0023: ミニバッファ補完機能

## ステータス

承認済み

## コンテキスト

switch-buffer, find-file, save-buffer でミニバッファ入力時に
Tab キーで補完できるようにしたい。

## 決定

### 補完インターフェース

`Completer` を関数型インターフェースとして core に定義する。
入力文字列を受け取り、補完候補のリストを返す。

### InputPrompter の拡張

`prompt(String, Completer)` オーバーロードを default メソッドとして追加する。
既存の `prompt(String)` との後方互換性を維持する。

### 補完ロジック（Tab 1回）

- 候補 0 件: 何もしない
- 候補 1 件: 入力を候補で置換
- 候補複数: 最長共通プレフィックスまで補完

### 具体的な Completer

- `BufferNameCompleter`: BufferManager のバッファ名で前方一致
- `FilePathCompleter`: ファイルシステムのディレクトリ一覧で前方一致
  - ディレクトリ一覧取得は関数型インターフェースで外部注入（副作用の外部化）

### スコープ外

- 連続 Tab での候補一覧バッファ表示（分割ウィンドウ描画と合わせて別タスク）

## 結果

- ミニバッファで Tab キーにより補完が効くようになる
- 補完ロジックは Completer の差し替えで柔軟に対応可能
