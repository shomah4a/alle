# ADR 0065: ミニバッファアクティブ時のother-windowウィンドウ巡回にミニバッファを含める

## ステータス

提案

## コンテキスト

現在の `Frame.nextWindow()` はウィンドウツリー内のウィンドウのみを巡回対象としている。
ミニバッファウィンドウは巡回対象に含まれていないため、以下の問題がある:

- ウィンドウ分割なし: ミニバッファ入力中にother-windowを実行しても移動先がない(ツリー内ウィンドウが1つのため早期リターン)
- ウィンドウ分割あり: ミニバッファからツリー内ウィンドウに移動できるが、ミニバッファには戻れない

ミニバッファが非アクティブの場合はミニバッファを巡回対象に含める必要はない。

## 決定

`Frame.nextWindow()` において、`minibufferActive` が `true` の場合にミニバッファウィンドウを巡回リストに追加する。

巡回順序はウィンドウツリーの深さ優先順の末尾にミニバッファウィンドウを配置する。
これはEmacsの挙動と同様である。

## 影響

- ミニバッファ入力中にother-windowでツリー内ウィンドウとミニバッファを自由に行き来できるようになる
- ミニバッファ非アクティブ時の挙動は変更なし
- Frame.nextWindow() メソッドのみの変更で済む
