# ADR-0042: ディレクトリ補完時の末尾スラッシュ追加

## ステータス

承認済み

## コンテキスト

find-file, save-buffer のミニバッファでファイルパス入力時にタブ補完を行う際、
ディレクトリに補完された後に再度タブを押しても、ディレクトリの中身を探索できない問題がある。

原因は `FilePathCompleter.complete()` が `Path.of(input)` でパスを正規化する際に
末尾の `/` が除去されるため、常に親ディレクトリから自身を探す動作になることにある。

## 決定

`FilePathCompleter.complete()` に、入力が `/` で終わる場合の分岐を追加する。
末尾が `/` の入力に対しては、そのパスをディレクトリとみなし、中身を一覧して返す。

これにより、ディレクトリへの補完完了後に再度タブを押すとディレクトリ内のエントリが
候補として表示される、Emacs と同様の挙動が実現される。

## 影響

- 変更対象: `FilePathCompleter.complete()` のみ
- `DirectoryLister`, `CompletionResult`, コマンド側への変更は不要
