# ADR 0110: GitHub Actions CI の導入

## ステータス

承認済み

## コンテキスト

本プロジェクトにはCIが存在せず、PRのビルド・テスト・lint検証が手動に依存している。
今後Dependabotによる依存更新の自動化を見据えており、その前提としてCIの整備が必要となる。

## 決定

### ジョブ構成

- ビルド・テストジョブとlintジョブを分離し、並列実行する
  - buildジョブ: `./gradlew build`（コンパイル + errorprone/nullaway + テスト）
  - lintジョブ: `./gradlew spotlessCheck`（フォーマットチェック + ワイルドカードインポートチェック）

### トリガー

- `push`(main), `pull_request`(main)

### 環境

- Java 21
- `gradle/actions/setup-gradle` によるキャッシュ

### セキュリティ

- `permissions: contents: read` で最小権限

### 効率化

- `concurrency` 設定で同一PRの古いジョブをキャンセル

### コスト

- パブリックリポジトリのため、GitHub Actions の分数制限・課金は発生しない

## 結果

- PRに対するビルド・lint検証が自動化される
- Dependabot導入の前提条件が整う
