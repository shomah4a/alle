# ADR 0039: 入力スレッドとロジックスレッドの分離

## ステータス

承認

## コンテキスト

現状は全処理が単一スレッドで動作しており、描画中はキー入力を受け付けられない。
ADR 0038 で描画スナップショットの分離が完了しているため、
スレッド分離の基盤が整った。

Lanterna はGUIフレームワークのようなイベントループを持たないため、
UIスレッドにLanterna操作を集約する必要はない。
readInput() と setCharacter()/refresh() は別スレッドから呼べる。

## 決定

### スレッド分離

- **入力スレッド（メインスレッド）**: キー入力の読み取りのみを担当
- **ロジックスレッド**: コマンド処理・スナップショット作成・描画を担当

### スレッド間通信

- `LinkedBlockingQueue<KeyStroke>`: 入力スレッド → ロジックスレッドにキーを渡す
- それ以外のスレッド間共有データはなし

### 入力スレッドのループ

1. `readKeyStroke()` でブロッキング待機
2. キーをキューに投入

### ロジックスレッドのループ

1. キューからキーを取り出し（ブロッキング）
2. `processKey()` でコマンド実行
3. `createSnapshot()` でスナップショット作成
4. `renderSnapshot()` で画面描画
5. `refresh(DELTA)` で画面更新

### 終了処理

- `requestShutdown()` が呼ばれると `readKeyStroke()` が empty を返す
- 入力スレッドが POISON_PILL をキューに投入
- ロジックスレッドが POISON_PILL を検知して終了

## 影響

- Frame/Window/Buffer へのアクセスはロジックスレッドに閉じる
- ScreenRenderer はロジックスレッドが独占使用（同期不要）
- 入力スレッドはキーの読み取りとキュー投入のみ
