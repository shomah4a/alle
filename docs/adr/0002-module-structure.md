# ADR-0002: モジュール構成

## ステータス

承認済み

## コンテキスト

エディタの各関心事を適切に分離し、独立して開発・テスト可能にする必要がある。
また、エディタ固有でない汎用的な機能は再利用可能なライブラリとして切り出したい。

## 決定

Gradleマルチプロジェクト構成で以下のように分割する。

### libs/

エディタに依存しない汎用ライブラリ。目的・機能ごとにサブモジュールを作る。

- `libs/gap-buffer/` — Gap Bufferデータ構造

### allei-*

エディタ固有のモジュール。

- `allei-core/` — バッファ、ウィンドウ等のドメインモデル
- `allei-command/` — コマンドシステム
- `allei-mode/` — メジャー/マイナーモード
- `allei-render-api/` — レンダリング抽象層(interface)
- `allei-tui/` — TUI実装
- `allei-scripting/` — 拡張言語
- `allei-app/` — エントリポイント、組み立て

## 帰結

- libsの汎用ライブラリはエディタ外でも利用可能
- 各モジュール間の依存方向が明確になる
- 必要なモジュールだけを段階的に実装できる
