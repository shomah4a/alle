# ADR 0093: tree-diredのディレクトリまたぎcp/mvにバリデーション追加

## ステータス

承認済み

## コンテキスト

tree-diredはツリー表示により複数ディレクトリのファイルを1つのバッファに表示する。
マークはPath（フルパス）のSetで管理されているため、異なるディレクトリ配下のファイルを自由にマークできる。

cp/mvコマンドで複数ファイルを操作する場合、宛先ディレクトリに対して各ファイルの`getFileName()`で配置先を解決する。
このため、異なるディレクトリのファイルをマークしてcp/mvすると、元のディレクトリ構造が失われてフラットに配置されるという意図しない動作になる。
また、異なるディレクトリに同名ファイルがある場合は上書きが発生する可能性がある。

delete、chmod、chownなどは各ファイルに独立して作用するため、ディレクトリをまたいでも問題ない。

## 決定

cp/mvコマンドの実行時に、マーク対象が2件以上かつ親ディレクトリが複数ある場合はエラーメッセージを表示して処理を中断する。

- バリデーションはresolveTargets直後、プロンプト（宛先入力）前に行う
- エラー時にマークはクリアしない
- バリデーションロジックはTreeDiredEntryResolverにstaticメソッドとして配置する
- マーク機能自体には制限を加えない（delete等は影響なし）

### 検討した選択肢

- A. マーク対象を同一ディレクトリに制限する → deleteなど問題ない操作にまで制限がかかるため却下
- B. cp/mv実行時にバリデーションする → 採用
- C. 確認メッセージを出す → フラット配置という結果自体は防げないため却下
- D. マーク複数時はcp/mvを無効化 → 同一ディレクトリ内の一括操作もできなくなるため却下

## 影響

- TreeDiredCopyCommand、TreeDiredRenameCommandにバリデーション呼び出しを追加
- TreeDiredEntryResolverにバリデーション用staticメソッドを追加
