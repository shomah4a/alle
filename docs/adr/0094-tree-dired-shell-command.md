# ADR 0094: tree-diredにシェルコマンド実行機能を追加

## ステータス

承認済み

## コンテキスト

tree-diredにはcp/mv/delete/chmod/chownといったファイル操作コマンドが実装されているが、
任意のシェルコマンドをファイルに対して実行する手段がない。

Emacsのdired-do-shell-command (`!`) に相当する機能として、マーク済みまたはカーソル行のファイルのパスを
シェルコマンドに渡して実行する機能を追加する。

## 決定

### コマンド仕様

- キーバインド: `!`
- コマンド名: `tree-dired-shell-command`
- 対象ファイル: `TreeDiredEntryResolver.resolveTargets()` で解決（マーク済みまたはカーソル行）

### コマンド文字列の置換ルール

Emacsの仕様に準拠する:

1. コマンド文字列に `*` が含まれる場合: `*` を対象ファイルパスのスペース区切りリストに置換
2. コマンド文字列に `?` が含まれる場合: ファイルごとにコマンドを個別実行し、`?` をファイルパスに置換
3. どちらも含まない場合: コマンド末尾にファイルパスをスペース区切りで追加
4. `*` と `?` が同時に含まれる場合: エラーとして拒否

### ファイルパスのクォート

ファイルパスはシングルクォートで囲んでシェルに渡す。
パス中のシングルクォートは `'\''` にエスケープする。
これによりスペースや特殊文字を含むファイル名を安全に扱う。

### 実行方式

- `/bin/sh -c` 経由でシェルコマンドとして実行（パイプ・リダイレクト等を利用可能にするため）
- stdout と stderr は別スレッドで並行読み取りし、パイプバッファ満杯によるデッドロックを回避する
- 作業ディレクトリは対象ファイルの親ディレクトリとする

### 結果表示

- `*Shell Command Output*` バッファを作成し結果を表示
- stdout と stderr を色分け表示する（stderrには `FaceName.WARNING` を適用）
- 末尾にexit codeを表示

### 副作用の外部化

プロセス実行は `ShellCommandExecutor` インターフェースとして抽象化し、テスト時にスタブに差し替え可能とする。

## 結果

- 任意のシェルコマンドをdiredバッファから直接実行できるようになる
- プロセス実行がインターフェースで抽象化されているため、テスタビリティが確保される
